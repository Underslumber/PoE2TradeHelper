# Регулярные слепки рынка

## Цель

Собирать не только текущую цену выбранной категории в открытом UI, а регулярную историю всего stackable-рынка актуальной лиги: валюты, фрагменты, эмоции, эссенции, руны, омэны, Expedition/Ritual/Ultimatum и другие категории из `trade2/data/static`.

## Что входит в слепок

- Лига, категория, валюта оценки и статус рынка.
- Для каждой позиции: `best`, `median`, `volume`, `offers`, 7-дневное изменение, `sparkline`, источник данных и иконка.
- `status=any` по умолчанию: фильтр online-продавца не нужен для долгой истории рынка.
- `volume` из poe.ninja хранится как прокси спроса/активности.
- `offers` заполняется для fallback через `trade2/exchange`; в poe.ninja overview число лотов обычно недоступно.

Публичные endpoints не дают подтвержденный журнал закрытых сделок. Поэтому поле спроса нельзя трактовать как точное количество продаж: это рыночный прокси по агрегированному объему и доступным офферам.

## Частота

- Первые 48 часов после старта лиги: каждые 5 минут.
- После 48 часов: каждые 15 минут.
- Для этого у CLI есть `--league-start`: пока текущее время попадает в окно `--early-days 2`, используется `--early-interval-minutes 5`, затем `--interval-minutes 15`.
- Когда запущено FastAPI-приложение, фоновый сервис включен по умолчанию и делает слепки без открытого браузера.
- Сервис перечитывает `trade2/data/leagues` раз в 10 минут и выбирает первую актуальную PoE2 challenge-лигу, исключая Standard/Hardcore/SSF/private. Если GGG добавит новую trade-лигу первой в списке, сбор переключится на нее.
- Для `Runes of Aldur` заложен известный старт `2026-05-29T19:00:00+00:00`, поэтому ранний 5-минутный режим включится по фактическому времени старта.

## Хранение

Текущий слой пишет JSONL в `data/trade_rate_history.jsonl`. По текущим слепкам один category/target snapshot обычно занимает десятки килобайт; полный проход по stackable-категориям в одном target ориентировочно укладывается в сотни килобайт. Диск не главный риск, главный риск - качество источников и дисциплина запросов.

Дальше стоит добавить compaction:

- 0-7 дней: сырые 5/15-минутные слепки.
- 8-30 дней: почасовые OHLC/volume агрегаты.
- Старше 30 дней: дневные OHLC/volume агрегаты.

Агрегат лучше считать как OHLC по цене и сумму/максимум по активности, а не оставлять случайный один снимок за час или день.

## CLI

Одноразовый полный проход:

```powershell
python -m app.cli market-snapshots --league "Fate of the Vaal" --once
```

Команда собирает все stackable-категории из static data, пишет их в `data/trade_rate_history.jsonl` и печатает JSON-сводку результата.

Непрерывный сбор с частым режимом первые два дня:

```powershell
python -m app.cli market-snapshots --league "Fate of the Vaal" --league-start "2026-05-16T22:00:00+03:00" --currency-targets divine,chaos
```

Команда будет писать слепки каждые 5 минут в первые 48 часов после указанного старта и каждые 15 минут позже. Дополнительные `--currency-targets` нужны только для валютного benchmark; полный рынок по умолчанию собирается в `exalted`.

Проверить статус фонового сервиса в запущенном UI:

```text
GET /api/trade/market-snapshot-service
```

Ответ показывает текущую выбранную лигу, известные лиги, время последней проверки, время последнего сбора и последнюю сводку.

## Использование в графиках и ИИ

Для UI добавлен легкий endpoint по одной позиции:

```text
GET /api/trade/history/item?league=...&category=...&item_id=...&target=exalted&status=any&metric=price
GET /api/trade/history/item?league=...&category=...&item_id=...&target=exalted&status=any&metric=demand
```

Он отдает временной ряд без отправки всех строк всех слепков в браузер. Детальные графики цены и спроса в UI используют этот endpoint и могут брать больше точек истории.
