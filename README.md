# PoE2 Trade Helper

## Цель

Сделать локального помощника для торговли в Path of Exile 2, который помогает быстро находить потенциально выгодные сделки, обмены и предметы для перепродажи.

## Что уже есть

- MCP-слой в `mcp_server.py` для базовой работы с PoE2 `trade2`: лиги, static data, search, fetch и exchange.
- Новый FastAPI-интерфейс в `app/` поверх отдельного торгового слоя `app/trade2.py`.
- Единый источник версии в `app/version.py`; версия выводится под названием приложения в верхней панели UI и используется для cache-busting статических файлов.
- Live UI на `/`, который загружает актуальные лиги и категории из `trade2/data/*`, показывает список позиций и умеет обновлять цены выбранной категории; выбор лиги вынесен в верхнюю панель, а рыночные фильтры показываются только в торговых/аналитических вкладках.
- Для поддерживаемых категорий `get_category_rates` сначала использует прямой JSON endpoint poe.ninja, затем откатывается на `trade2/exchange`.
- Нормализация static entries, exchange listings и poe.ninja overview в единый формат строк.
- Вкладка сигналов показывает рыночную сводку по poe.ninja: падения/росты за 7 дней, объем и риск низкой ликвидности.
- 7-дневные `sparkline.data` из poe.ninja exchange overview трактуются как процентный ряд и перед показом переводятся в ряд цен, привязанный к текущей цене позиции.
- В рыночной сводке есть скоринг лучших сделок: действие, оценка приоритета, риск, объем и позиция цены в 7-дневном диапазоне.
- Сигналы используют внутреннюю историю снимков для сравнения с прошлым снимком без показа raw-лога на странице.
- Поиск публичных priced-лотов конкретного продавца через `trade2/search/fetch` и сравнение с похожими buyout-лотами рынка по типу предмета, уровню, официальным `extended.hashes`/stat id и нормализованным аффиксам.
- Stackable-лоты продавца из поддерживаемых категорий оцениваются быстрее через агрегированную цену poe.ninja; редкие/уникальные предметы остаются на сравнении похожих лотов через `trade2`.
- Кэш снимка публичных лотов продавца, чтобы повторные фильтры и сравнения не выгружали весь список заново.
- Локальная регистрация и авторизация с подтверждением email: пользователи, сессии, закрепленные позиции и журнал сделок хранятся в SQLite.
- Есть отдельная вкладка админки: админ видит список учеток, может выдавать доступ к админке/ИИ и смотреть локальные метрики ИИ-ресурса.
- Подтверждение email умеет отправлять письмо через SMTP из `.env`; если SMTP не настроен, локальный UI показывает dev-ссылку подтверждения.
- Личный кабинет на live UI разделяет отслеживание с графиком, открытые сделки и закрытые сделки: профильная валюта расчета применяется к новым закреплениям/входам, вход фиксирует цену и benchmark, открытая сделка показывает текущую номинальную и реальную маржу, выход сохраняет итоговую маржу.
- Реальная маржа считается относительно выбранного эталона покупательной способности, сейчас это Divine/Exalted/Chaos из валютного рынка. Формула: рост позиции делится на рост benchmark за тот же период, чтобы видеть не только прибыль в валюте, но и изменение покупательной способности.
- В профиле зарегистрированного пользователя можно включить скрытый рублевый слой FunPay: backend парсит только публичную витрину `https://funpay.com/chips/209/`, сохраняет снимки в SQLite, строит почасовой график низкого рынка и показывает рублевые ориентиры валюты, запас, продавцов и прокси движения лотов без пересчета маржи в рубли.
- В кабинете можно создать Telegram-уведомления по закрепленным позициям: цена выше/ниже порога, изменение в процентах или любое изменение цены; отправка срабатывает при обновлении цен категории.
- В личном кабинете есть короткая памятка по поведению рынка на старте лиги: фазы свежей экономики, риски ранних цен, ликвидность и benchmark-подход.
- Черновой инструмент для ИИ собирает JSON-контекст рынка через `app.ai_context`, API `/api/ai/market-context` и CLI `python -m app.cli ai-context`.
- У пользователя с правом `can_use_ai` появляется вкладка ИИ в live UI: она запускает backend-задачу `/api/ai/market-analysis`, показывает статус, сводку, сигналы, недостающие данные и путь к сохраненному аудиту.
- Веб-API ИИ требует входа в учетку с правом `can_use_ai` или админкой; вызовы пишутся в локальные метрики и ограничиваются `AI_DAILY_QUOTA`. CLI остается локальным инструментом.
- Codex CLI-анализатор рынка запускается через `python -m app.cli market-analyze`: он берет тот же JSON-контекст, вызывает `codex exec` в read-only режиме, валидирует JSON-ответ и сохраняет входной payload/ответ/итоговую оценку в `data/ai_market_analyses`.
- Валютный анализатор `app.currency_analyzer` / `/api/trade/currency-analysis` / `python -m app.cli currency-analyze` строит историю выбранной валюты, локальный тренд, волатильность и осторожный прогноз; из вкладки ИИ можно передать этот контекст в Codex через `/api/ai/currency-analysis` и увидеть отдельный совет.
- Фоновый CLI-сборщик `python -m app.cli market-snapshots` делает регулярные слепки всех stackable-категорий актуальной лиги в `status=any`: 5 минут в первые 48 часов при заданном `--league-start`, затем 15 минут.
- При запуске FastAPI-приложения фоновый сервис слепков включен по умолчанию: он выбирает актуальную PoE2 trade-лигу, собирает market snapshots и раз в 10 минут проверяет появление новой лиги.
- Тем же фоновым циклом собирается рублевая база FunPay для Divine: один публичный снимок `chips/209` на цикл, сохранение в `funpay_rub_snapshots` / `funpay_rub_offers`, настройка через `FUNPAY_RUB_SNAPSHOT_ENABLED` и `FUNPAY_RUB_SNAPSHOT_TARGET`.
- Для графиков одной позиции есть легкий API `/api/trade/history/item`, чтобы читать длинный ряд цены или спроса без отправки всех строк всех слепков в браузер.
- Черновой расчет 3-в-1 цепочек Liquid Emotions: сравнение `3 x исходная эмоция -> следующая эмоция` по медианной цене.
- Внутренняя история снимков live-курсов в JSONL для анализа и API, без отдельного видимого лога на странице.
- Старый collector для poe.ninja с discovery/sync, SQLite-хранилищем, иконками, CSV/JSONL export и страницей `/economy`.
- Набор быстрых unit-тестов для нормализации trade2/poe.ninja данных, парсера строк и placeholder-иконок.

## Что не доделано

- Нет полноценной модели прибыльности: пока учитывается только простая разница цен, без spread, свежести, объема, фейковых listings, price fixing и практической исполнимости.
- Для фрагментов, проходок и boss-entry предметов пока есть только вывод цен и заметка, без расчета рецептов/составов.
- История курсов пишется в JSONL, но нет нормального анализа трендов, сравнения снимков и сигналов по изменению рынка.
- Нет compaction старой истории: нужны почасовые агрегаты после 7 дней и дневные агрегаты после 30 дней.
- Анализ вставленного текста предмета из игры еще не реализован: нужен parser/normalizer, затем отдельный pricing engine.
- Анализ выставленных игроком товаров не реализован; для PoE2 пока нет подтвержденного официального пути к личным stash данным.
- Поиск лотов продавца работает только по публичным trade2 listings и не заменяет доступ к личным stash данным.
- README пока не описывает пользовательский сценарий запуска и ограничения UI достаточно подробно.
- Рублевый слой FunPay считает изменения запасов и числа лотов только как прокси спроса/предложения: это не подтвержденные сделки и не стакан исполненных покупок. Завышенные цены отсекаются из ориентира по низу рынка, но фейковые дешевые listings все еще требуют ручной проверки.

`mcp_server.py` логирует внешние вызовы без тел запросов и секретов. Ошибки API возвращаются в виде словаря с `error` и реальным `status_code`, если сервер его прислал.

## Запуск

Установка зависимостей в уже созданное локальное окружение:

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Команда ставит зависимости проекта из `requirements.txt` в `.venv`.

Запуск нового локального UI:

```powershell
python -m app.cli run
```

Команда поднимает FastAPI-приложение на `http://localhost:8000`.
Пока приложение запущено, фоновый сервис market snapshots активен по умолчанию. Его статус доступен через `/api/trade/market-snapshot-service`.
Если в `.env` задан `ADMIN_PASSWORD`, при старте будет создана или обновлена учетка `ADMIN_USERNAME` с правами админа и ИИ. Если в базе нет админов, первый существующий или первый зарегистрированный пользователь тоже становится админом. Вкладка админки показывается только админам, вкладка ИИ - только админам и пользователям с `can_use_ai`. `AI_DAILY_QUOTA` задает локальную дневную квоту веб-ИИ; `0` отключает локальный лимит.

Одноразовый сбор слепков рынка:

```powershell
python -m app.cli market-snapshots --league "Fate of the Vaal" --once
```

Команда собирает все stackable-категории через poe.ninja/trade2 и пишет историю в SQLite (`data/poe2_ninja.sqlite`, таблица `market_history`). Старый `data/trade_rate_history.jsonl` используется только как источник миграции и fallback для legacy-запусков.

Анализ рынка через Codex CLI:

```powershell
python -m app.cli market-analyze --league "Fate of the Vaal" --category Currency --target exalted --status any --league-day 1
```

Команда берет последний сохраненный снимок категории, строит JSON-контекст для ИИ, запускает `codex exec` в read-only режиме и печатает нормализованную оценку. Результат аудита сохраняется в `data/ai_market_analyses`. Если добавить `--refresh`, перед анализом будет сделан свежий запрос к рынку.

Анализ одной валюты:

```powershell
python -m app.cli currency-analyze --league "Fate of the Vaal" --currency divine --target exalted --status any --league-day 1 --horizon-hours 24
```

Команда читает сохраненные снимки Currency, считает историю, направление тренда, волатильность и локальный прогноз. Если добавить `--with-ai`, тот же контекст будет передан Codex CLI в read-only режиме для текстового совета.

Непрерывный сбор для старта лиги:

```powershell
python -m app.cli market-snapshots --league "Fate of the Vaal" --league-start "2026-05-16T22:00:00+03:00" --currency-targets divine,chaos
```

Команда использует `status=any`, собирает основной рынок в `exalted`, дополнительно пишет валютные benchmark-снимки в `divine` и `chaos`, а частоту выбирает по окну старта: 5 минут первые 48 часов, 15 минут дальше.
