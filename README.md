# PoE2 Trade Helper

Локальный веб-помощник для торговли в Path of Exile 2. Он собирает рыночные данные, показывает историю цен, помогает искать осторожные торговые идеи и вести собственный журнал сделок.

Проект не обещает автоматическую прибыль и не покупает предметы сам. Его задача - дать проверяемые данные, расчеты и контекст, чтобы решение оставалось за игроком.

## Для чего

- Быстро смотреть рынок актуальной лиги: валюты, фрагменты, эмоции, эссенции, руны и чистые предметные основы.
- Сравнивать цены из live-listings и сохраненной истории, а не по одному случайному снимку.
- Оценивать торговые сигналы с учетом spread, объема, свежести данных и риска price fixing.
- Анализировать рецепты, обменные маршруты и цепочки преобразований.
- Проверять публичные buyout-лоты продавца и сравнивать выбранные предметы с похожим рынком.
- Вести личный кабинет: закрепленные позиции, входы и выходы из сделок, комиссии, стратегии, Telegram-уведомления и P/L.
- Передавать ИИ только подготовленный рыночный контекст, без права выдумывать цены или выполнять сделки.

## Что внутри

- `app/` - FastAPI-приложение, веб-интерфейс, API и сервисный слой.
- `app/web/` - UI, шаблоны и статические файлы.
- `app/trade2.py` и `app/trade/` - работа с PoE2 trade2, rate limits и рыночными расчетами.
- `app/market_snapshots.py`, `app/market_service.py`, `app/history_compaction.py` - сбор, хранение и сжатие истории рынка.
- `app/account.py`, `app/db/`, `app/notifications.py` - кабинет, SQLite, сессии и уведомления.
- `mcp_server.py` - standalone MCP-сервер для доступа к trade2.
- `scripts/` - вспомогательные скрипты для каталога предметных основ и локальных proxy-туннелей.

## Источники данных

- PoE2 trade2: live-поиск, fetch, exchange и справочники торговли.
- poe.ninja: агрегированные цены для поддерживаемых stackable-категорий.
- PoB PoE2 + PoE2DB: seed-каталог чистых normal-предметных основ и русские названия; будущие классы вроде мечей, кинжалов, кастетов и топоров исключаются, а фоновый сбор сначала проходит увеличенным batch по метовым слотам poe.ninja и снижает приоритет уже увиденных основ дешевле `1 exalted`.
- SQLite: локальная история `market_history`, пользователи, сделки, закрепленные позиции и служебные данные.
- FunPay: скрытый пользовательский рублевый слой через публичную витрину, если он включен в профиле.

Для обычного `trade2/search`, `trade2/fetch` и `trade2/exchange` API-ключ обычно не нужен. Нужен корректный `USER_AGENT`. Официальный `https://api.pathofexile.com` использует OAuth и отдельные scopes.

## Запуск

```powershell
.\.venv\Scripts\python.exe -m pip install -r requirements.txt
```

Устанавливает зависимости проекта в локальное виртуальное окружение.

```powershell
.\.venv\Scripts\python.exe -m app.cli web
```

Запускает FastAPI-интерфейс на локальном порту из `PORT` или на значении по умолчанию.

## Конфигурация

Основные настройки лежат в `.env`; публичный шаблон - в `.env.example`.

Ключевые переменные:

- `USER_AGENT` - идентификатор клиента для внешних API.
- `PORT`, `APP_BASE_URL` - локальный запуск веб-приложения.
- `PUBLIC_CANONICAL_ORIGIN`, `PUBLIC_API_ORIGIN`, `PUBLIC_REDIRECT_HOSTS` - внешний origin UI/API, если приложение опубликовано наружу.
- `OUTBOUND_PROXY_URL`, `POE2_PROXY_URL`, `OUTBOUND_PROXY_URLS`, `POE2_PROXY_URLS` - исходящий proxy/VPN для trade2, poe.ninja, PoE2DB, FunPay, Telegram и MCP-вызовов.
- `MARKET_SNAPSHOT_*` - фоновый сбор рыночных снимков.
- `FUNPAY_RUB_SNAPSHOT_*` - рублевый слой FunPay.
- `SMTP_*`, `TELEGRAM_BOT_TOKEN` - подтверждение email и уведомления.

Секреты и локальные значения хранятся только в `.env`; их нельзя коммитить.

## Ограничения

- Рыночные выводы зависят от качества listings, свежести данных, spread, объема и поведения продавцов.
- `volume` в сохраненной истории - прокси активности, а не точное число закрытых сделок.
- Длинные rate limits trade2 не обходятся агрессивным polling: приложение должно показывать сохраненные данные или частичный результат.
- Личный stash-анализ PoE2 не считается надежным, пока для него не будет заново проверенного официального пути.
