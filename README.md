# POE2 Trade Helper

Инструментарий для подключения к торговым API Path of Exile 2, получения данных о листингах и быстрой аналитики по цене, валюте и продавцам. Есть CLI, который принимает готовый JSON‑запрос (search или exchange), подсчитывает статистику и по желанию отправляет сводку в локальный LLM.

## Возможности
- Быстрые запросы к публичным эндпоинтам `search` и `exchange` c извлечением первых N листингов.
- Подсчёт средней/медианной цены, минимального и максимального значения, популярных валют и продавцов.
- Выгрузка данных в человекочитаемом виде для ручного анализа или последующей обработки.
- Генерация промпта и отправка статистики на локальный LLM HTTP‑эндпоинт (например, llama.cpp, LM Studio и др.).

## Установка зависимостей
Проект использует Python 3.11+ и требует только `requests`:

```bash
python -m venv .venv
source .venv/bin/activate
pip install --upgrade pip
pip install requests
```

## Пример использования CLI
1. Подготовьте JSON‑файл с запросом, например `currency_query.json`:
```json
{
  "exchange": {
    "status": { "option": "online" },
    "have": ["chaos"],
    "want": ["divine"]
  }
}
```
(Структура соответствует официальному PoE trade API и может быть экспортирована с сайта.)

2. Запустите CLI:
```bash
python -m poe2tradehelper.cli currency_query.json --league standard --exchange --limit 40
```

3. (Опционально) Добавьте локальный LLM для вербальной сводки:
```bash
python -m poe2tradehelper.cli currency_query.json --exchange --llm-endpoint http://localhost:11434/api/generate
```

## Архитектура
- `poe2tradehelper.client.PoE2TradeClient` — низкоуровневый клиент с удобными методами `search_items` и `exchange_currency`, а также отдельными вызовами для получения «сырых» ответов и детализированных листингов.
- `poe2tradehelper.analytics.ListingAnalytics` — слой агрегации: работа с ценами, топ‑валюты, продавцы, выборки по полям айтема.
- `poe2tradehelper.llm.LlmInsights` — генерация промпта и отправка статистики на любой совместимый HTTP‑эндпоинт LLM.
- `poe2tradehelper.cli` — минимальная CLI‑обвязка над клиентом и аналитикой.

## Идеи для расширения
- Кэширование результатов, чтобы не упираться в rate limit официального API.
- Хранение истории цен (например, в SQLite) с графиками и ежедневными срезами.
- Асинхронные запросы для одновременной загрузки большего числа листингов.
- Автоматические алерты по заданным фильтрам и связка с дискорд‑ботом.
- Визуализация (Streamlit/Gradio) для быстрой ручной работы.
