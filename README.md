# PoE2 Trade Helper

Локальный помощник для торговли в Path of Exile 2. Проект помогает смотреть рынок, сравнивать цены, находить осторожные торговые сигналы и вести собственные сделки без обещаний автоматической прибыли.

## Цель

Собрать практичный инструмент, который отвечает на три вопроса:

- что сейчас выглядит дешево или дорого относительно рынка;
- какие обмены, рецепты и цепочки могут дать маржу с учетом рисков;
- как меняется личный результат игрока после входа и выхода из сделки.

## Что хотим получить

- Быстрый обзор актуальной лиги: валюты, фрагменты, эмоции, эссенции, руны, другие stackable-категории и чистые предметные основы.
- Историю цен и спроса, чтобы видеть тренд, а не только один случайный снимок рынка.
- Осторожные сигналы покупки, продажи и обмена с учетом spread, объема, свежести listings и риска price fixing.
- Анализ рецептов и многошаговых преобразований, где вход и выход можно посчитать как цепочку.
- Поиск всех публичных buyout-лотов продавца с локальной фильтрацией выдачи, оценкой похожего рынка и отдельным анализом дорогих крафтовых основ.
- Профиль продавца по умолчанию и ручной выбор основы, важных/игнорируемых свойств, уровня аффикса и диапазона значений для сравнения выбранного лота.
- Оценку вставленного текста предмета из игры через parser, normalizer и похожие публичные listings.
- Личный кабинет для закрепленных позиций, входов/выходов из сделок, Telegram-уведомлений и истории результата.
- Рыночный контекст для ИИ, где модель объясняет риски и идеи, но не подменяет расчет цен.

## Уже реализовано

### Рыночный обзор

- FastAPI-интерфейс с выбором лиги, категорий, валюты оценки и деталями позиции.
- Публичный интерфейс использует канонический origin `https://xapct.ru:9038`; заходы на `https://xapct.ru` без порта перенаправляются на порт, чтобы браузер не терял сессию кабинета и закрепленные позиции между разными origin. API на `https://xapct.ru/api/...` остается доступным с CORS для `https://xapct.ru:9038`, чтобы браузерные account-запросы не ломались из-за `Alt-Svc`/HTTP3 на публичном порту.
- Получение live-данных через trade2 и агрегированных цен через poe.ninja с fallback между источниками.
- Исходящие HTTP-запросы могут идти через системный прокси или локальный VPN/proxy-клиент: `OUTBOUND_PROXY_URL` / `POE2_PROXY_URL` применяются к trade2, poe.ninja, PoE2DB, FunPay, Telegram и MCP-вызовам. Для нескольких endpoints есть `OUTBOUND_PROXY_URLS` / `POE2_PROXY_URLS`: режим `OUTBOUND_PROXY_STRATEGY=failover` держит активный endpoint и при сетевой ошибке или настроенном ответе о блокировке/недоступности повторяет запрос через следующий endpoint, `round_robin` чередует endpoints. `429` не считается поводом для переключения по умолчанию, чтобы сохранять дисциплину `Retry-After`.
- Для SSH SOCKS-туннелей к выделенным VPS есть `scripts/start_proxy_tunnels.ps1` и `scripts/stop_proxy_tunnels.ps1`; они поднимают локальные endpoints `127.0.0.1:1080` и `127.0.0.1:1081` без изменения системных маршрутов.
- Bootstrap-справочники лиг и категорий с локальным fallback, чтобы временный сбой trade2 не оставлял интерфейс без валют и разделов.
- Автоматический фоновый сбор market snapshots по stackable-категориям и `ItemBases` при запуске приложения; список чистых normal-основ берется из PoB PoE2 (`D:\Soft\PathOfBuilding-PoE2\Data\Bases`) через `scripts/build_item_base_catalog_seed_from_pob.py`, а русские названия добираются из PoE2DB. Генератор сохраняет tracked seed в `app/resources/item_base_catalog_seed.json` и локальный runtime-снимок `data/item_base_catalog.json`, выкидывая `DNT/UNUSED`, `not_for_sale`, `hidden`, синтетические варианты в скобках, флаконы, самоцветы, удочки, ядра душ, trap tools и прочий не-equipment мусор. Общий рынок основ строится из этого каталога и дозаполняется небольшими порциями по первым 10 priced-лотам каждой основы, но UI/API показывают только основы с найденными clean priced лотами или явной ошибкой сбора: пустые будущие позиции без рыночного подтверждения не выводятся. Рыночный поиск основ ходит через английский endpoint `www.pathofexile.com/api/trade2`, потому что PoB-каталог отправляет английский `type`, а видимые русские названия остаются из seed/PoE2DB. HTTP-слой trade2 соблюдает `X-Rate-Limit-*` / `Retry-After`, чтобы не продолжать сбор поверх активного ограничения. Точный сбор первых 100 лотов включается только для фильтра/уточнения. Иконки основ кэшируются локально в `storage/icons/item-bases`, а fallback-иконки генерируются как локальные SVG.
- Чтение последних сохраненных снимков из SQLite без частого браузерного polling.
- История raw/hourly/daily с compaction для старых данных.
- Пользовательские цены в интерфейсе округляются до сотых сферы/валюты без длинных технических дробей.

### Торговая аналитика

- Сводка рынка по изменению цены, объему, ликвидности и положению в диапазоне.
- Разделение рекомендаций на сильные сигналы, слабые сигналы и наблюдение.
- Диагностика качества данных: свежесть снимка, покрытие цен, рисковые строки и проверка сигналов по прошлым снимкам.
- Анализ исполнения: spread, свежесть снимка, число лотов, объем и риск тонкого рынка.
- Кросс-валютные маршруты `A -> B -> ... -> A` по фактическим online-офферам.
- Recipe-framework для составных предметов, полная логика путей Liquid Emotions, splinter-to-entry цепочки Breach/Ritual/Expedition и расчет стоимости fragment/boss-entry комплектов.

### Предметы и продавцы

- Поиск публичных priced-лотов продавца через trade2.
- Кэш снимка лотов продавца, чтобы повторные фильтры не выгружали рынок заново.
- Сравнение предметов по типу, редкости, item level, official stat id и fallback-аффиксам.
- Ручной профиль выбранного лота: основу можно учитывать отдельно, важные official stat id становятся обязательными, лишние исключаются, а для аффикса можно зафиксировать tier/level или диапазон roll-значений.
- Режим оценки крафтовой основы без свойств, топ дорогих основ продавца и отдельный рынок чистых normal-баз: сначала показывается сохраненный каталог основ, затем фоном собираются первые 10 мгновенных priced-лотов по каждой основе, а для выбранной/отфильтрованной основы уточняются первые 100 лотов, с исходной валютой лота, пересчетом через сохраненные Currency-снимки, нижней ценой, оптимумом по первой сотне, историей нижней цены и переключаемой сортировкой по столбцам.
- Быстрая оценка stackable-лотов продавца через poe.ninja, если позиция точно совпала с поддерживаемой категорией.
- Parser вставленного текста предмета из игры и осторожная оценка pasted-item по похожим public listings.

### Личный кабинет

- Локальные пользователи, сессии, подтверждение email и админские права.
- Продавец по умолчанию в профиле для быстрого повторного поиска своих или отслеживаемых лотов.
- Отдельная вкладка отслеживания чистых основ в кабинете; основа добавляется из рынка основ и обновляется по сохраненным `ItemBases`-снимкам.
- Закрепленные позиции с графиком, открытые сделки и закрытые сделки.
- Фиксация входа, выхода, комиссии, тега стратегии и причины сделки.
- Benchmark-adjusted P/L через Divine, Exalted, Chaos или basket:liquid-core.
- Сводный отчет по сделкам: номинальный и benchmark-adjusted P/L по валютам, комиссии, win rate и разбивка по стратегиям.
- Telegram-уведомления по цене, процентному изменению, повышенному спросу по основе или любому изменению закрепленной позиции.
