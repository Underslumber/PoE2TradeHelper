# Требования к реализации

Нужно заменить временные patch/workflow-артефакты на реальные изменения исходников.

1. `app/trade2.py`
   - Добавить `ITEM_BASE_MARKET_MIN_ILVL = 78`.
   - Заменить старый порог `ITEM_BASE_MARKET_LOW_PRIORITY_MAX_EXALTED = 1.0` на `ITEM_BASE_MARKET_REFRESH_MIN_EXALTED = 10.0`.
   - Все запросы рынка чистых основ (`_item_base_market_query`, `_item_base_market_overview_query`, точный и фоновый сбор) должны использовать `min_ilvl >= 78`.
   - Если пользователь передал `min_ilvl < 78` или `None`, использовать `78`; если передал больше — сохранить это значение.
   - Старые rows с `min_ilvl is None` не считать покрытием диапазона 78+.
   - Основы без сохранённой цены оставлять в первичном rough-обходе.
   - Основы с ценой `< 10 exalted` исключать из повторных фоновых запросов полностью.
   - Основы с ценой `>= 10 exalted` оставлять в обновлении и приоритетном recheck.

2. `app/web/routes.py`
   - Для `/api/item-bases/market` использовать дефолтный `min_ilvl=78`.
   - При запуске refresh job передавать этот `min_ilvl`, не `None`.

3. Тесты
   - Проверить ilvl floor 78 для query builders.
   - Проверить, что legacy rows без `min_ilvl` не проходят фильтр 78+.
   - Проверить, что неизвестная основа остаётся в первичном обходе.
   - Проверить границу 9.9/10 exalted: 9.9 пропускается, 10.0 обновляется.
   - Проверить несколько циклов: основы дешевле 10 не возвращаются в выборку.

4. Документация и версия
   - README: описать `item level 78+` и повторное обновление только от `10 exalted`.
   - `AGENTS.md`: закрепить это правило.
   - Версия `0.2.109 -> 0.2.110`, тест версии обновить.

5. Убрать из PR все временные артефакты:
   - `*.patch`
   - `notes/*`
   - `.github/workflows/apply-item-base-policy.yml`

Проверки перед финальным коммитом:

```bash
python -m compileall mcp_server.py app
python -m pytest -q
```
